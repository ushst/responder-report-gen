# Responder Report Gen

Локальный (без бэкенда) HTML/JS-инструмент, который воссоздаёт вид консольного
вывода [Responder](https://github.com/lgandx/Responder) — строки перехвата
NTLMv1/NTLMv2 (и других типов авторизации: `Basic`, `Cleartext`, `APOP`,
`CRAM-MD5`, `DIGEST-MD5`, `PLAIN-SASL`, `NetNTLMv1/v2`), а также
poison-сообщения LLMNR / NBT-NS / MDNS — для использования в пентест-отчётах,
когда оригинальный скриншот перехвата потерян, а сами данные (IP, hostname,
username, хэш) остались в логах.

Инструмент **ничего не перехватывает и никуда не отправляет данные** — это
чисто визуальный реконструктор. Все значения хранятся только в `localStorage`
твоего браузера.

## Откуда взят формат

Формат строк не выдуман, а сверен построчно с исходниками Responder:

- `utils.py` → `SaveToDb()` — шаблоны строк `Client   :` / `Hostname :` /
  `Username :` / `Password :` / `Hash     :`, а также функции `color()` /
  `text()`, определяющие раскраску (`[MODULE]` — жирный синий, значение —
  жёлтый).
- `servers/SMB.py`, `servers/HTTP.py`, `servers/MSSQL.py`, `servers/LDAP.py`,
  `servers/RDP.py`, `servers/RPC.py`, `servers/WinRM.py`, `servers/IMAP.py`,
  `servers/POP3.py`, `servers/SMTP.py`, `servers/FTP.py`, `servers/MQTT.py`,
  `servers/Proxy_Auth.py` — какие типы (`type`) авторизации реально
  встречаются на каждом модуле.
- `poisoners/LLMNR.py`, `poisoners/NBTNS.py`, `poisoners/MDNS.py` — точный
  формат poison-строк (включая двойной пробел после `[LLMNR]` — это не баг
  генератора, так выглядит оригинал).

## Использование

Открой `index.html` в браузере (двойной клик или `xdg-open index.html`) —
никакой сборки/сервера не требуется.

1. Слева добавляй строки: **Poison** (LLMNR/NBT-NS/MDNS), **Захват хэша/логина**
   (модуль + тип авторизации подбираются автоматически под реальные комбинации
   Responder), произвольные строки, пустые строки.
2. Заполняй поля своими реальными данными.
3. Справа — живой предпросмотр окна терминала (3 темы, размер шрифта,
   заголовок окна).
4. Делаешь обычный скриншот области экрана средствами ОС/браузера.

Дополнительно:
- **Экспорт/импорт JSON** — сохранить весь сценарий в файл и переиспользовать
  позже или на другом устройстве.
- **Автосохранение** — состояние сохраняется в `localStorage`, обновление
  страницы ничего не теряет.
- **Скопировать как текст** — если для отчёта нужен именно текст, а не
  картинка.

## Хостинг

Статический сайт без зависимостей — можно открывать локально по `file://`
либо раздать через GitHub Pages / любой статический хостинг.

## Дисклеймер

Инструмент предназначен для оформления отчётов по легитимному пентесту /
red team, когда перехват уже был выполнен авторизованно, а исходный
скриншот утерян. Он не выполняет ARP/LLMNR/NBT-NS spoofing и не работает с
сетью — это визуальный шаблонизатор.
